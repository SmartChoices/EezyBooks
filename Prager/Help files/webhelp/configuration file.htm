The Configuration file

We use a configuration file to give the program information it needs when starting up that cannot be obtained by other means.  It is named Inventory.cfg and is found in the same directory as where you installed the program.  You can use a normal text editor (Notepad works nicely) to modifyt it if necessary.  The contents look like this:

<Configuration>
<!-- don't change the following three lines unless instructed to do so  -->
    <ServerInstance>(local)</ServerInstance>
    <ServerInstallationPath>C:\Program Files\Microsoft SQL Server\MSSQL.1\MSSQL\Data\</ServerInstallationPath>
    <DatabasePath>C:\Prager\</DatabasePath>

<!--  the following three lines may be changed if needed  -->
    <BackupPath>c:\Prager\Backup\</BackupPath>  <!--  in which directory do you want your backup files placed? -->
    <ExportPath>c:\Prager\Export\</ExportPath>  <!--  in which directory do you want your export files placed? -->
    <DaysRetention>30</DaysRetention>  <!--  how many days do you want to keep backup and export files?  -->
</Configuration>

ServerInstance and ServerInstallationPath are for compatability with previous versions.  The DatabasePath is the location of the Firebird database, normally in c:\Prager.

The other lines are self-explanatory.  Note that 14 days retention assumes you backup your database daily.  You can extend this if necessary; it is only used to "prune" the directory so as to avoid having thousands of unused files taking up space.
