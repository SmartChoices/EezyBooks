Tracing

Setup

Occasionally, there comes a time when we need to trace the inner workings of the program.   This topic is meant to show you how to turn "tracing" on and off.  First, you need to look for a file called Prager Book Inventory.exe.config which you will find in the installation directory (normally c:\Program Files\Prager).  Once you have found that file, double-click on it to open it... you will get a pop-up window that looks something like this in Figure 61. 

Figure 61.  Open Dialog

 

When you click on the "Choose Program" menu item, you get a screen that looks like the following in Figure 61a.

Figure 61a.  The "Open With" dialog window

 

Scroll down until you see the listing for Notepad and click on it (that selects it)... click OK and you will be presented with the following, as shown in Figure 61b.

Figure 61b.  The Prager Book Inventory.exe.config file opened in Notepad.

 

Notice the two highlighted (in yellow and pointed to by the red arrows)... these two lines need to be removed; however, we don't want to delete the lines, but rather just overwrite them using the space bar.  This will have the effect of removing the characters which are directions to comment out the lines between the highlighted lines.  The lines in between the highlighted lines cause the program to generate a trace file called pragerTrace.txt. 

Once you have removed the lines by overwriting them, save the file by clicking on the Main Menu->File->Save.  Now, run the Prager Inventory program and when it stops (due to an error or ?), send us the file pragerTrace.txt as an attachment to an email to support@pragersoftware.com.  Once this is done, you need to add those lines we removed back in in the exact place they were.  This turns off tracing, which takes a lot of resources and will definitely slow the program.  Don't forget, put the characters back in the same positions exactly as shown: first line has a left-arrow, exclamation point followed by two dash symbols (<!--)... the second line is two dash symbols followed by a right arrow (-->).

We will examine the trace file and hopefully be able to diagnose the problem; if we don't have enough information, you might be asked to download a special version which has more tracing in it.
