Uploading records to listing sites

Setting your User IDs and Passwords

Figure 13a.  This is where you set your User IDs and Passwords.  Once you have entered them, you must save them; you can over-ride them at any time, but you should always save your changes.  Some important notes:

  • Columns 4-6 for the pre-defined sites can not be altered. 
  • Amazon.com requires your email address for the User ID field.  In addition, you now have to have an "Amazon Access Key" and an "Amazon Secret Key" in order to get prices and bibliographic data for each book.  Click on the Get Keys button to go to the Amazon website, get your keys and copy and paste them into the respective boxes.
  • In order for you to upload to Half.com, you must have a "security token", which can be obtained from Half.com.  

If you have listing venues that are not listed here, you can add them using the Custom Listing Sites.  To use the custom sites feature, you need to fill in the information as indicated above.  Some of this information will have to come from the listing service itself (FTP address and directory).  Once this has been filled in, you can go to the Upload tab where you will see the "Name" of the custom site indicated.  Just check the box and the export file chosen will be uploaded to that service.  It is recommended that you test your information that was entered by checking the "Test" box and doing a test upload first.  Note that this is only for FTP uploads; some sites do now allow FTP uploads, so you should check with them to make sure it is a feature they support.

 

One-Click uploading to listing services

 

Figure 13.

First of all, and this is very important: next to some of the listing sites, you may see a '2'.  This means that you must register with the listing services to be able to upload using FTP, which is what this program uses.  Simply contact them and ask for a FTP account.  They will respond via e-mail giving you a new user ID and password.  This is the user ID and password you must use for uploading.

Also next to the listing sites is the format of the file that is sent to that particular site.  If the site gives you a choice of formats, pick the format we have indicated we use.  If you don't pick the correct format, your uploads will not be accepted by the listing site.

Next to the Upload button, it tells you if an export  file is waiting to be uploaded since your last upload.  There are ninteen (19) sites you can currently upload your inventory changes to as seen above.  Note that some of them are disabled (lighter color)... this is because if you don't have a userid and password, you can not choose to upload to that site. 

To upload to any or all of the sites, do the following:

  1. Create an export file,
  2. go to the Upload tab (see figure 38 above).
  3. Make sure you have a UserID and Password set for the service you are going to upload to.  If you do not have the UserID and Password set for a particular site will not be enabled (text will be dimmed).
  4. check the box next to the listing service you want to upload to
  5. enter your user name and password for that site (it will automatically be saved when you exit the program)
  6. click on "Save Selections" if you want to keep the selected sites between executions of the program 
  7. click "Upload" and you will be presented with the Open Dialog box as seen below. 

Note the list of filenames all begin with HB... this is because the program knows which file-type goes with which listing venue.  You should always choose from this list; do not enter your own filename because it may not be found when the name is changed for other listing venues.  For instance, if you want to upload to Amazon.com, it takes a .txt file; find the HB prefixed filename from the list above and click it; the filename will automatically be changed for Amazon.com within the program.  Note:  if you do not have an ISBN or an ASIN for a particular record, it will not be uploaded.  You are going to have to do it manually.  This applies to Amazon.com uploads only!

  • just do an Inclusive search: select ISBN, "is equal to" and leave the remainder of the fields blank. 
  • Then go to the export page and click on "Only those displayed in the Database Panel using Inclusive Search". 
  • Click export and a file with only those records without ISBNs will be created. 
  • Take this file and manually submit it using the Standard Book Loader.

It is recommended you test your first uploads by clicking the "Test" checkbox.  This has the effect of not uploading a real file, but a file that has been designed to test the connection and your UserID and Password.  The file name is found at  c:\HBTestUpload.txt; this file should be edited to change items that are specific to your business (line USER|Prager located at the top of the file).  There is also a file in UIEE format called UITestUpload.txt.  It should also be edited.

 You can follow progress of the upload(s) in the Status Panel.  The file format (HomeBase, UIEE or Tab) is automatically selected depending on listing site requirements.  You will always be presented with a HB prefix file to select in the Open Dialog.  This is normal; the program uses that selection to find the correct file and construct new filenames accordingly.

NOTE:  you can not do an automatic purge on all listing sites using the automated upload at this time.  This is because in most cases, the listing web-site has a box to click or some other action that must be done to effect the purge.
