Restoring the database

There is only one way to restore a corrupted database, and that is to click Database->Restore.  You are given a choice of which backup file to use; just choose the one you want as seen in the figure below.

and click Open in the Open Dialog box. Everything else is done for you automatically.

 

Doing a "restore" from the Command Prompt

    The first three (3) steps are the same as the backup; for a restore of the badkup file to the database, but I included them here anyway.

  • Step 1 - go to the Start Menu and click on Start -> Accessories -> Command Prompt

  • Step 2 - you will see the Command Prompt window open

  • Step 3 - It is positioned at your default directory. We need to change this, so do the following:
      • where the c:\Users\Rolf> is, type the following:
      • cd \
      • cd \"Program Files" or cd \"Program Files (x86)" if you are on a 64-bit machine (don't forget the double-quotes ( " ))
      • then type the following: cd \Firebird\bin
      • and it should look like this (notice that what you should have typed in is highlighted in yellow):

  • Step 4 - we are going to type in the following statement and press Enter on your keyboard:

        • gbak.exe -REP -user sysdba -pass masterkey "C:\Prager\Backup\B20120203150911.gbk" C:\Prager\dbBooks.fdb
                        ^         ^         ^             ^          ^                  ^                                                                              ^

        where what's in red is the filename of the backup file that you created, found in C:\Prager\Backups and the ^ indicate where spaces are in the command.

        Step 5 - after you press Enter,you should see the same thing as in Step 5 above, except for the command, it will start with -REP (if there were any errors, you should take a screen shot of the Command Prompt window and send it to support |at| pragersoftware |dot| com):
