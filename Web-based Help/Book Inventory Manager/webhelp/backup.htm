Backing up the database

 

Backing up the database

There are 2 ways to backup the database:

1.  by clicking on Backup database when exiting as shown in the following figure...

or 2.  by clicking on Database->Backup as shown in the figure below.

They both accomplish the same thing, but at a different time in the processing.  Clicking on Database->Backup is immediate; clicking on Options->Backup database when exiting occurs when you exit the program.  At any time you need to rebuild the database, just restore the latest backup file.

All backup files are kept in the directory C:\Prager\Backup.   They have a name made up of the database name (in this case dbBooks) fllowed by the date (mmddyyyy) and time (hhmmss) and an extension of .bak.  Days retention refers to the amount of days the file is retained before being deleted.  Fourteen (14) days is the default (see configuration file for how to change this).  NOTE:  sometimes there is a problem while running under Windows XP where it gives you an error message saying "Cannot backup... Access Denied", which occurs when you have your backup path set to c:\program files\prager.  This is a bug of Microsoft's and the workaround is to select a different Backup Path such as the default (C:\Prager\Backup).

 

Doing a backup from the Command Prompt

There are rare times that you need to do a backup of the database from a Command Prompt. This is how you do it.

Step 1 - go to the Start Menu and click on Start -> Accessories -> Command Prompt

Step 2 - you will see the Command Prompt window open

Step 3 - It is positioned at your default directory. We need to change this, so do the following:

where the c:\Users\Rolf> is, type the following:

cd \

cd \"Program Files" or cd \"Program Files (x86)" if you are on a 64-bit machine (don't forget the double-quotes ( " ))

then type the following: cd \Firebird\bin

and it should look like this (notice that what you should have typed in is highlighted in yellow):

Step 4 - we are going to type in the following statement and press Enter on your keyboard:

gbak.exe -backup -user sysdba -pass masterkey C:\Prager\dbBooks.fdb c:\Prager\Backup\B20120212125718.gbk
                ^              ^         ^             ^          ^                  ^                                         ^

where what's in red is the filename of the backup that consists of a "B" for books, the date in the format yyyymmdd followed by the time as HHMMSS and ends with ".gbk" and the ^ indicate where spaces are in the command.

Step 5 - after you press Enter, this is what you will see (if there were any errors, you should take a screen shot of the Command Prompt window and send it to support |at| pragersoftware |dot| com):
