Creating a file for export

Indicate which export files to create: (New)

You must indicate to the program which venues you are going to upload to during the export process.  Some of the venues (i.e. ABE, Amazon, Chrislands, etc) require a special formatted file.  To save space on your hard drive, the program will NOT create an export file for those venues unless you tell it to, by clicking on the venue name in the top box.  If the venue name is checked, a file for that venue will be created, otherwise it will not.  If you don't upload to a particular venue, don't check the box.

General Exporting instructions:

There are two times when you might want to perform an export of your records: 

  1. when you are going to do a purge and replace of all of your records on a listing site, or
  2. when you are informing the site that you had activity on your inventory (books added, sold, etc.) by uploading to the site

Both are accomplished the same way:

  • Export files are always created using HomeBase, UIEE, comma-separated values (CSV) and tab-delimited formats, because some listing services require one or the other. (you don't have to worry about which file goes with which listing service, it is done automatically for you by the program).  Tab-delimited files are created in two possible formats: normal tab-delimited and Amazon.com-specific formats.  
  • if you want to export all of your records, click "Export entire database".   NOTE:  At the end of the export, a message box will pop-up and will ask you if you are doing a purge/replace of the entire inventory.  This affects the date "Changed Since"  date which needs to be changed to match the exported file.  This is done automatically by the program if you click Yes. (otherwise, you can really screw things up because the program won't know you did a purge and will again export all the records that have been changed since the date indicated). 
  • If you want to only export a sub-set of records that you have selected using the Inclusive Search feature, check "Only those displayed...".  What ever is showing in the Database Panel will then be included in the export file.
  • if you only want to export the records that have changed since the last export, select "Changed since" button.

It is assumed that all of your user ID's's are identical for each of the listing services.  The file that is exported has a User ID in the header; by default this ID is taken from the first user ID on the Upload Tab page for the appropriate file type.

The export files are created automatically and placed in the C:\Prager\Export folder with the name looking like this:   "FFmmddyyyyhhmmss.txt"  where

  • FF is either HB for HomeBase format, UI for UIEE format, CV for comma-separated value, TB for normal tab-delimited format or  AZ for Amazon.com . 
  • mmddyyyy represents the date of the export
  • hhmmss represents the time of the export.

NOTE:  If you don't see the filename you expect, it is because there were no records to export, therefore the file was not created.

Records are only marked for export when added using the Add or Update buttons; they are not marked for export when added to the database using the Import function.  An indication of how many records exported is displayed next to Records Processed:.  The filename is also displayed for ease in finding it for uploading.

ABE file format conversion:

Recently, ABE has indicated they prefer a tab-delimited format rather than the HomeBase format they have used for years.  When you do an export, you will be asked if you want to change the format from HomeBase to tab-delimted.  If you have already done the conversion, click on the "Don't ask me this again" checkbox, and you won't see the reminder mesage.

If you reply no, then your export to ABE will be in the HomeBase format.  If you reply yes, then several things will occur:

  • First, click on the checkbox highlighted in yellow in the image above.  This will create a tab-delimited export file for ABE with an extension of HOLD.  This will notify ABE that your file needs to be verified.
  • Next, you must notify ABE that you have sent them a file for uploading with an extension of HOLD.  They will verify the file format and then process it.
  • Once both of the above have been accomplished, and ABE has notified you that your file has been accepted, you can then ignore the message about "Have you notified ABE ..." when you do the export.
